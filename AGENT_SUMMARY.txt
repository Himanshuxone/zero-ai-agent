================================================================================
                        DATA FLOW & ORCHESTRATION
================================================================================

FLOW DIAGRAM:

    User Dashboard
           ↓
    [Ticker Selection]
    (Agent 2 Action)
           ↓
    Orchestrator Store
    (Central State Hub)
           ↓
    ┌─────────────────────────────────────────────────────┐
    │ Agents Execute in Parallel                          │
    ├─────────────────────────────────────────────────────┤
    │ Agent 1: Fetch /api/stocks/live        (30s)       │
    │ Agent 3: Fetch /api/portfolio          (On demand) │
    │ Agent 4: Fetch /api/fundamentals       (On demand) │
    │ Agent 5: Display trending stocks       (30s)       │
    └─────────────────────────────────────────────────────┘
           ↓
    Store Updated with New Data
           ↓
    React Components Re-render
           ↓
    Updated Dashboard Display


STATE MANAGEMENT FLOW:

Orchestrator Store manages:
  ├─ selectedTicker: "AAPL" (user selection from Agent 2)
  ├─ marketData: [{price: 185.42, change: +1.29%, ...}] (Agent 1 data)
  ├─ holdings: [{ticker: "AAPL", quantity: 100, ...}] (Agent 3 data)
  ├─ companyInfo: {pe: 32.5, marketCap: 2.9T, ...} (Agent 4 data)
  └─ Error states and loading flags for each agent


DATA REFRESH CYCLES:

Agent 1 (Market Data):     Every 30 seconds (live data)
Agent 3 (Portfolio):       On demand (static user data)
Agent 4 (Fundamentals):    On demand (company data)
Agent 5 (Trends):          Every 30 seconds (trending data)

Caching Strategy: SWR (Stale-While-Revalidate)
  • Client-side cache: 60 seconds for charts, 30 seconds for live data
  • Fallback to cached data if network unavailable
  • Revalidation on window focus for fresh data
